# 대화 기반 개인 지식 도서관

LLM과 나눈 대화가 채팅창 안에서 사라지지 않고, 다시 읽고 복습할 수 있는 문서로 남도록 만든 개인용 지식 저장소다.

이 프로젝트는 [Andrej Karpathy의 글](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)를 읽다가 아이디어를 얻어 만들었다. LLM과 대화를 주고받다 보면 분명히 많이 배우는데, 세션이 끝나면 그 학습 내용들이 그대로 유실되는 것이 늘 아깝게 느껴졌다. 그래서 그 대화들을 흘려보내지 않고, 나만의 wiki처럼 정리하고 관리할 수 있으면 좋겠다는 생각으로 이 저장소를 만들었다.

이 저장소는 앱 소스코드 프로젝트라기보다, 대화에서 나온 배움과 개념을 축적하는 개인 도서관에 가깝다. 핵심 아이디어는 단순하다. 대화를 하고, `정리해줘`라고 말하면, 그날 공부한 내용을 먼저 읽기 좋은 리포트로 남기고, 정말 다시 볼 가치가 있는 개념만 library 문서로 축적한다. 한 세션 안에 큰 주제가 둘 이상이면 리포트도 하나로 뭉개지 않고 주제별로 나눠 남긴다.

## 왜 이런 구조인가

보통 LLM과의 대화는 세션이 끝나면 흩어진다. 로그를 그대로 보관해도 나중에 다시 읽기엔 길고 산만하다. 이 저장소는 그 문제를 해결하기 위해 다음 원칙으로 운영된다.

- 대화 원문은 보관한다.
- 그날 공부한 내용은 `report/` 아래의 줄글 리포트로 다시 읽기 좋게 정리한다.
- 한 세션에 큰 주제가 둘 이상이면 `report/`도 주제별로 여러 개 만들 수 있다.
- 반복해서 다시 볼 핵심 개념만 `library/`에 남긴다.
- 필요하면 더 다듬은 글을 `writing/posts/`에 남긴다.
- `library/index.md`, `library/log.md`로 축적 과정을 추적한다.

## 기본 흐름

1. 대화 원문을 `inbox/`에 둔다.
2. 대화를 충분히 진행한다.
3. 마지막에 `정리해줘`라고 말한다.
4. 그날 공부한 내용이 `report/` 아래의 리포트로 정리된다.
5. 필요한 경우에만 `library/` 문서가 생성되거나 업데이트된다.

## 디렉터리

- `inbox/`: 아직 정리되지 않은 대화 원문
- `archive/`: 정리 완료 후 보관하는 원본 기록
- `report/`: 오늘 공부한 내용을 다시 읽기 위한 학습 리포트
- `library/`: 선택적으로 축적하는 핵심 개념, 주제, 세션 문서
- `writing/posts/`: 위키 기반 파생 글
- `system/`: 운영 가이드와 템플릿
- `scripts/`: 도서관 유지용 로컬 도구

## 호출 예시

- `정리해줘`: 현재 대화를 학습 리포트로 정리
- `위키화해줘`: 개념/주제 문서를 적극 반영
- `글로 써줘`: 읽기 좋은 글로 다시 작성
- `질문으로 남겨줘`: 아직 답이 덜 된 질문을 보관

세부 규칙은 [system/guide.md](/C:/Users/user/working/llm-wiki/system/guide.md)에 정리되어 있다.

## 빠른 시작

대화 원문 파일이 있을 때 새 세션 초안을 만들려면:

```powershell
.\scripts\library_cli.ps1 new-session -Title "예시 세션" -Date 2026-04-13 -SummaryFile .\inbox\example.md
```

인덱스를 다시 만들려면:

```powershell
.\scripts\library_cli.ps1 rebuild-index
```

작업 로그를 추가하려면:

```powershell
.\scripts\library_cli.ps1 log -Type ingest -Title "예시 세션" -Details "session note 생성"
```

## 운영 원칙 요약

- `정리해줘`의 기본 산출물은 `report/` 아래의 줄글 리포트다.
- `library/`는 모든 개념을 잘게 쪼개 넣는 곳이 아니다.
- 이 저장소는 "리포트 우선, 라이브러리 선택적 축적" 방식으로 운영한다.
