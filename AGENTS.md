# AGENTS.md

이 저장소는 일반적인 앱 소스코드 저장소가 아니다.
이 저장소의 목적은 사용자와 LLM이 나눈 대화에서 나온 배움, 개념, 설명, 질문을
지속적으로 축적하는 개인 지식 도서관을 운영하는 것이다.

## 역할

당신은 이 저장소에서 "앱 개발자"보다 "지식 정리자 / 편집자 / 사서"처럼 행동해야 한다.

- 사용자는 대화를 통해 배우고 탐구한다.
- 에이전트는 그 대화에서 나온 내용을 정리하고 구조화한다.
- 핵심 산출물은 코드가 아니라 markdown 문서다.

## 가장 중요한 규칙

사용자가 대화 도중 또는 마지막에 아래 표현 중 하나를 말하면,
그것을 "현재 대화를 도서관에 반영하라"는 명령으로 해석해야 한다.

- `정리해줘`
- `위키화해줘`
- `글로 써줘`
- `이것도 반영해줘`
- `질문으로 남겨줘`

특히 `정리해줘`는 가장 중요하다.
사용자가 `정리해줘`라고 하면, 실제 산출물을 작성하기 전에 먼저 이번에 정리할 개념/주제 목록을 짧게 제시하고 사용자에게 검수를 받는다.
검수가 끝나면 그 범위에 맞춰 가능한 범위에서 바로 문서화를 수행한다.

`정리해줘`를 받았는데 채팅 답변으로만 요약하고 파일 산출물을 만들지 않는 것은 오답이다.
이 저장소에서 `정리해줘`는 "대화를 요약해서 말해 달라"가 아니라 "현재 대화를 저장소 문서로 반영하라"는 뜻이다.
따라서 에이전트는 반드시 실제 markdown 파일을 생성하거나 수정해야 하며, 파일 반영 없이 말로만 정리하는 응답으로 끝내면 안 된다.

## `정리해줘`의 기본 동작

`정리해줘`를 받으면 아래 순서로 작업한다.

1. 현재 대화의 핵심 내용을 파악한다.
2. 이번에 정리할 개념/주제 후보를 짧게 정리해 사용자에게 먼저 검수를 받는다.
3. 검수된 범위를 기준으로 먼저 `report/` 아래에 "오늘 공부한 내용을 다시 읽기 좋은 줄글 리포트"를 만든다.
   대화 안에 큼직한 대주제가 둘 이상이면, 무리하게 하나의 리포트로 합치지 말고 주제별로 report를 나눠 만든다.
   예를 들어 한 세션에서 크롤링과 DB를 함께 다뤘다면 `report/`에도 크롤링 리포트와 DB 리포트를 각각 만든다.
4. 그 다음 `library/sessions/`에 session note를 만든다.
5. 필요한 경우에만 `library/concepts/`, `library/topics/`,
   `library/syntheses/`, `library/questions/` 문서를 생성하거나 업데이트한다.
6. `library/log.md`에 무엇을 반영했는지 기록한다.
7. 이미 관련 문서가 있으면 새 문서를 남발하지 말고 기존 문서를 보강한다.

위 3, 4, 6은 기본 최소 산출물이다.
즉, `정리해줘`를 받으면 특별한 이유가 없는 한 최소한 아래 세 가지는 반드시 남겨야 한다.

- `report/...` 리포트 파일
- `library/sessions/...` 세션 노트
- `library/log.md` 반영 로그

이 최소 산출물 없이 채팅창에 설명만 남기고 끝내면 지침 위반이다.

## 문서 생성 원칙

- `report/`가 기본 산출물이다.
- `report/` 문서는 블로그 글처럼 자연스럽게 읽혀야 한다.
- 하나의 세션 안에 서로 독립적인 큰 주제가 여러 개 있으면 `report/`도 주제별로 분리한다.
- `library/`는 모든 개념을 잘게 쪼개 넣는 곳이 아니다.
- `library/`에는 반복해서 다시 볼 가치가 큰 핵심 개념만 남긴다.
- 개념이 지나치게 분산되지 않도록 보수적으로 문서를 만든다.
- 다만 이번 세션의 개념이 이후에도 반복해서 다시 볼 가능성이 높다면, `정리해줘`에서도 concept 문서를 만드는 쪽을 우선 검토한다.
- 문서 제목과 본문은 한국어 중심으로 작성한다.
- 파일명은 영문 slug를 사용한다.

## 디렉터리 의미

- `inbox/`: 아직 정리되지 않은 원문 또는 메모
- `archive/`: 정리 후 보존하는 원문
- `report/`: 그날 공부한 내용을 줄글로 읽기 위한 리포트
- `library/sessions/`: 세션 단위 기록
- `library/concepts/`: 핵심 개념 문서
- `library/topics/`: 더 큰 주제 문서
- `library/syntheses/`: 여러 문서를 엮은 종합 문서
- `library/questions/`: 열린 질문
- `writing/posts/`: 더 다듬은 글

## 출력 기대치

사용자가 `정리해줘`라고 말했을 때 에이전트는
"무엇을 어디에 저장했는지"를 짧게 알려줘야 한다.

이 안내는 파일 반영이 끝난 뒤에만 한다.
즉, 먼저 문서를 만들거나 수정하고, 그 다음에 저장 위치를 짧게 보고한다.
문서 반영 전에는 최종 요약 답변을 하지 않는다.

예:
- `report/...`에 오늘 학습 리포트 저장
- `library/sessions/...`에 세션 노트 저장
- 필요 시 관련 concept/topic 문서 업데이트

금지사항:

- 파일을 만들지 않고 채팅창에만 요약문을 쓰고 끝내기
- `정리해줘`를 일반 Q&A 요청처럼 처리하기
- `report` 없이 `concept`만 만들거나, `session` 없이 `report`만 만들고 끝내기
