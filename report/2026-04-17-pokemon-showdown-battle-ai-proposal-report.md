# Pokemon Showdown 배틀 AI 프로젝트 제안 보고서

## 1. 프로젝트 개요

- 프로젝트명: `Pokemon Showdown 배틀 AI`
- 프로젝트 성격: `설명 가능한 배틀 의사결정 엔진`
- 한 줄 정의:
  - Pokemon Showdown 로그를 바탕으로 현재 배틀 상태를 해석하고
  - 시즌 메타 기반 확률 추정을 활용해
  - 선출과 턴별 행동을 추천하는 시스템

## 2. 문제 정의

- Pokemon Showdown 배틀은 단순 타입 상성만으로 판단하기 어렵다.
- 실제 판단에는 아래 요소가 동시에 들어간다.
  - 공개된 기술
  - 남은 HP
  - 상태이상
  - 교체 가능성
  - 엔트리 해저드
  - 속도 관계
  - 비공개 아이템 / 기술 / 테라 가능성
- 초보자는 메타 정보를 알아도 현재 턴 판단으로 연결하기 어렵다.
- 따라서 필요한 것은 정보 검색기가 아니라 `현재 상태 기반 판단 시스템`이다.

## 3. 프로젝트 정체성

- 이 프로젝트는 `RAG 프로젝트`가 아니다.
- 이 프로젝트는 `단순 코칭 서비스`도 아니다.
- 핵심은 `배틀 AI / 의사결정 엔진`이다.
- 메타 정보는 핵심 기능이 아니라 `추천 엔진의 재료`다.
- 설명 기능은 본체가 아니라 `추천 신뢰도 보강 장치`다.

## 4. 목표

- 특정 시즌 / 특정 포맷 하나를 대상으로 한다.
- 배틀 시작 전 선출 추천을 제공한다.
- 배틀 중 턴별 행동 추천을 제공한다.
- 추천과 함께 짧은 근거 설명을 제공한다.
- 실제 래더에서 규칙 기반 베이스라인보다 높은 성능을 목표로 한다.

## 5. MVP 범위

### 포함

- 특정 시즌의 특정 포맷 1개
- Showdown 로그 파싱 기반 상태 추적
- 선출 추천
- 턴별 행동 추천
- 공개 정보 + 시즌 메타 기반 확률 추정
- 추천 + 근거 출력
- 실제 래더 기반 평가

### 제외

- 여러 시즌 / 여러 포맷 동시 지원
- 팀 빌딩 추천
- 복기 / 리플레이 해설
- 자동 플레이
- 자동 입력
- 비공개 정보를 사실처럼 단정하는 강한 추정

## 6. 입력 방식

- 입력 방식: `Pokemon Showdown 로그 파싱`
- 채택 이유:
  - 자연어 입력보다 상태 구조화가 일관적임
  - 공개 정보 기준점이 분명함
  - 추천 로직과 입력 파이프라인을 분리하기 쉬움
- 제외 이유:
  - 자연어 입력은 상태 누락 위험이 큼
  - 표현 차이로 인한 해석 오차가 큼
  - 승률 중심 평가에 불리함

## 7. 추천 로직

- 현재 공개 정보를 구조화한다.
  - HP
  - 상태이상
  - 공개 기술
  - 교체 여부
  - 엔트리 해저드
  - 남은 포켓몬 상태
- 시즌 메타 정보를 사용해 비공개 정보에 확률을 부여한다.
  - 기술 세트
  - 아이템
  - 테라 타입
- 가능한 행동 후보를 평가한다.
  - 공격
  - 교체
  - 테라 사용 여부
- 가장 적절한 행동을 추천한다.

## 8. 의사결정 원칙

- 비공개 정보는 `확률 기반 가정`으로만 다룬다.
- 불확실성이 크면 `보수적인 선택`을 우선한다.
- 평가가 비슷하면 `안전한 선택`을 우선한다.
- 추천에는 근거를 함께 붙인다.

### 근거 설명 예시

- `상대 망나뇽은 스케일샷/신속/지진 세트 가능성이 높아서 지금은 강철 타입 유지가 안정적입니다.`

## 9. 검증 방식

- 핵심 평가지표: `실제 래더에서 규칙 기반 베이스라인보다 높은 성능`
- 비교 대상:
  - 타입 상성
  - 남은 HP
  - 교체 안정성
  - 단순 우선순위 규칙
  - 위 요소만 사용하는 규칙 기반 에이전트
- 성공 조건:
  - 제안 시스템이 규칙 기반 베이스라인보다 일관되게 높은 성능을 보임
- 실패 조건:
  - 성능 차이가 없거나 오히려 낮음

## 10. 팀 프로젝트 적합성

- 역할 분담이 가능하다.
- 예시 분담:
  - 상태 추적 파이프라인
  - 메타 데이터 수집 / 정리
  - 확률 추정 및 추천 엔진
  - UI / 시연 흐름 / 설명 출력
- 따라서 단순한 "LLM 붙인 서비스"보다 팀 프로젝트 구조를 만들기 쉽다.

## 11. 예상 리스크

- 실제 래더 평가는 외부 변수가 많다.
- 시즌 / 포맷을 넓히면 범위가 급격히 커진다.
- LLM 역할과 규칙 엔진 역할이 섞이면 설계가 흐려질 수 있다.
- 메타 기반 추정이 약하면 추천 품질이 흔들릴 수 있다.

## 12. 강사 컨펌용 핵심 메시지

- 이 프로젝트는 `메타 검색기`가 아니다.
- 이 프로젝트는 `불완전 정보 하에서 행동을 추천하는 배틀 AI`다.
- 핵심 난제는 검색이 아니라 아래 4가지다.
  - 상태 추적
  - 비공개 정보 확률 추정
  - 행동 추천
  - 성능 검증
- 따라서 보고서와 발표에서도 `AI 추천 서비스`보다 `배틀 의사결정 엔진`으로 설명하는 것이 적절하다.
