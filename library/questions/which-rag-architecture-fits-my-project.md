# 내 프로젝트에서는 어떤 RAG 구조가 맞을까

## 왜 중요한가

- 이론을 이해하는 것과 실제 프로젝트에 적용하는 것은 다르기 때문이다.

## 현재까지 얻은 단서

- 간단한 경우에는 Parent-Child나 Multi-Vector로도 충분할 수 있다.
- 긴 문서와 전체 맥락이 중요하면 RAPTOR가 적합할 수 있다.
- 관계 탐색과 multi-hop reasoning이 중요하면 Graph RAG가 더 잘 맞을 수 있다.
