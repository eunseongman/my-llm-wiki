# RAG는 검색보다 구조화 문제에 가깝다

## 핵심 논지

- 좋은 RAG의 성능은 단순히 검색기나 임베딩의 품질보다, 데이터를 어떤 구조로 조직하느냐에 더 크게 영향을 받는다.

## 충돌하거나 더 확인할 점

- 구조화 비용이 늘어날수록 구축 복잡도와 운영 비용도 함께 커진다.
- 모든 프로젝트가 RAPTOR나 Graph RAG 수준의 고급 구조를 필요로 하지는 않는다.
