# 운영 가이드

## 이 저장소의 목적

이 저장소는 사용자와 LLM의 대화에서 나온 지식을 구조화해 축적하는 개인 도서관이다. 대화 자체보다, 대화에서 뽑아낸 지식 문서가 중심 산출물이다.

## 기본 원칙

- 원본과 정리본을 분리한다.
- 같은 주제는 가능하면 기존 문서를 업데이트한다.
- 모든 중요한 문서는 다시 읽기 쉬운 문장으로 쓴다.
- 문서끼리 링크를 충분히 건다.
- 중요한 주장과 통찰은 세션 출처를 남긴다.
- 아직 답이 없는 질문도 버리지 말고 문서로 남긴다.
- `정리해줘`의 1차 산출물은 개념 쪼개기보다 "오늘 공부 리포트"여야 한다.
- 개념 문서는 정말 재사용 가치가 큰 경우에만 선택적으로 만든다.

## 디렉터리 규칙

- `inbox/`: 아직 정리되지 않은 원문
- `archive/`: 정리 완료된 원문
- `report/`: 그날 공부한 내용을 줄글로 다시 읽기 위한 리포트
- `library/sessions/`: 날짜별 또는 대화별 세션 기록
- `library/concepts/`: 개념 문서
- `library/topics/`: 큰 주제 문서
- `library/syntheses/`: 여러 문서를 엮은 종합 문서
- `library/questions/`: 열린 질문
- `writing/posts/`: 복습용 글, 에세이, 블로그형 문서

## 문서 작성 규칙

- 파일명은 영문 slug를 쓴다.
- 제목과 본문은 한국어 중심으로 쓴다.
- frontmatter에 최소 `title`, `type`, `status`, `created`, `updated`를 둔다.
- report 문서는 "내가 나중에 다시 읽을 복습 글"처럼 자연스럽게 읽혀야 한다.
- session 문서는 관련 concept/topic/question 문서로 링크한다.
- concept/topic 문서는 관련 session 문서 링크를 둔다.
- concept 문서는 아래 조건 중 하나를 만족할 때만 새로 만든다.
  - 앞으로 여러 번 재사용할 핵심 개념이다
  - 다른 주제에서도 반복 등장할 가능성이 높다
  - 독립된 문서로 따로 정리해야 오히려 이해가 좋아진다

## 권장 운영 흐름

### 대화 반영

1. 대화 원문을 `inbox/`에 저장한다.
2. 먼저 `report/` 아래에 그날 공부한 내용을 읽는 글로 정리한다.
3. session note를 만든다.
4. 필요한 경우에만 concept/topic/question 문서를 생성 또는 업데이트한다.
5. 가치가 큰 연결은 synthesis 문서로 승격한다.
6. `library/index.md`와 `library/log.md`를 갱신한다.
7. 원문을 필요하면 `archive/`로 옮긴다.

### 복습

1. 주제에 맞는 topic/concept 문서를 읽는다.
2. 관련 session과 synthesis를 따라간다.
3. 필요하면 새 synthesis 또는 writing 글을 만든다.

### 린트

정기적으로 아래를 확인한다.

- 고아 문서가 있는가
- 비슷한 개념이 중복 생성되었는가
- 최근 세션이 핵심 topic 문서에 반영되었는가
- 질문 문서가 방치되고 있지 않은가

## 대화 중 호출 규칙

앞으로 사용자는 고정된 명령어 대신 자연스러운 짧은 표현으로 도서관 업데이트를 요청할 수 있다.

기본 호출 규칙은 다음과 같다.

- `정리해줘`
  - 현재 대화를 먼저 `report/` 아래의 학습 리포트로 정리한다.
  - 그리고 session note를 만들고, 필요한 경우에만 관련 concept/topic/question 문서를 업데이트한다.

- `위키화해줘`
  - 현재 대화에서 나온 개념과 주제를 library 문서로 적극 반영한다.
  - session note를 넘어서 concept/topic/synthesis 연결까지 시도한다.

- `글로 써줘`
  - 현재 대화나 기존 library 문서를 바탕으로 `writing/posts/`에 읽기 좋은 글을 만든다.

- `이것도 반영해줘`
  - 직전 대화 내용을 기존 문서에 누적 업데이트한다.

- `질문으로 남겨줘`
  - 아직 답이 덜 된 포인트를 `library/questions/` 문서로 저장하거나 업데이트한다.

운영 원칙:

- 사용자가 위 표현 중 하나를 말하면, 별도 모드 전환 없이 바로 도서관 반영 작업을 수행한다.
- 사용자가 특별히 범위를 좁히지 않으면 기본적으로 현재 대화의 최근 문맥을 반영 대상으로 본다.
- 이미 존재하는 문서가 있으면 새 문서를 무조건 만들기보다 기존 문서를 보강하는 방향을 우선한다.
- `정리해줘`에서는 report를 먼저 만들고, library 문서는 과도하게 쪼개지지 않도록 보수적으로 생성한다.
- 반영 후에는 어떤 내용이 report, session, concept, topic, question, writing 중 어디에 저장되었는지 짧게 알려준다.

## 템플릿 사용

새 문서를 만들 때는 `system/templates/` 아래 템플릿을 복사해서 시작한다.

## 인덱스 규칙

`library/index.md`는 도서관의 입구다. 문서를 유형별로 나누고 각 문서의 한 줄 설명을 붙인다.

## 로그 규칙

`library/log.md`는 append-only로 유지한다. 각 항목은 날짜, 작업 유형, 제목, 간단한 설명을 포함한다.
