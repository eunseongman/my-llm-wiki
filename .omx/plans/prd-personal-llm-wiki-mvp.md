# PRD: 대화 기반 개인 지식 도서관

## 요약

이 저장소는 어떤 별도 "앱"을 만드는 공간이 아니라, 사용자와 LLM이 나눈 대화에서 추출된 지식이 누적되는 개인용 지식 도서관이다. 핵심 목표는 채팅 세션 안에서 사라지는 학습 내용을 이 폴더 안의 구조화된 마크다운 문서로 지속적으로 축적하고, 이후 다시 읽고 연결하고 복습할 수 있게 만드는 것이다.

## 문제 정의

사용자는 LLM과 대화하면서 많은 것을 배우지만, 그 내용은 보통 세션 안에서만 유효하고 시간이 지나면 흐려진다. 대화 로그를 그대로 쌓아 두는 것만으로는 복습과 종합이 어렵다. 필요한 것은 "대화 기록 보관함"이 아니라, 대화에서 뽑아낸 개념, 설명, 통찰, 비교, 열린 질문이 정리되어 축적되는 지식의 도서관이다.

## 목표

- 이 폴더 자체를 사용자-LLM 대화에서 추출한 지식의 지속적 저장소로 만든다.
- 대화에서 나온 중요한 내용을 위키형 문서로 정리해 나중에 다시 읽을 수 있게 한다.
- 필요할 때는 위키보다 더 읽기 쉬운 블로그형 정리 글도 파생 생성할 수 있게 한다.
- 새 대화가 기존 지식을 덮어쓰지 않고 보강, 수정, 연결하도록 한다.
- 사용자가 "예전에 이 주제로 뭐 배웠지?"를 이 폴더 안에서 탐색하며 답할 수 있게 한다.

## 비목표

- 일반 사용자를 위한 독립 실행형 제품이나 SaaS를 만드는 것
- 처음부터 UI, 계정 시스템, 서버, 배포 파이프라인을 구축하는 것
- 모든 대화를 자동 수집하는 무거운 인프라를 먼저 만드는 것
- 완전 자동화를 위해 사용자의 조향 가능성을 없애는 것

## 사용자 관점

- 사용자는 이 저장소를 자신의 "대화형 학습 도서관"으로 사용한다.
- LLM은 이 도서관의 사서이자 편집자 역할을 한다.
- 사용자는 질문하고, 생각을 발전시키고, 방향을 잡는다.
- LLM은 정리하고, 연결하고, 갱신하고, 기록한다.

## 핵심 원칙

- 폴더 우선: 결과물은 앱 내부 상태가 아니라 이 저장소의 파일로 남아야 한다.
- 지식 우선: raw 대화보다 정제된 지식 페이지가 중심 산출물이어야 한다.
- 누적 우선: 새 대화가 들어올 때마다 새로 쓰기보다 기존 페이지를 보강하는 방향을 우선한다.
- 출처 보존: 중요한 정리와 주장은 어떤 대화에서 나왔는지 추적 가능해야 한다.
- 재독성 우선: 나중에 사용자가 직접 읽고 복습하기 쉬워야 한다.

## 우리가 만드는 것

우리가 만드는 것은 크게 네 가지다.

1. 대화를 보존하는 원본 계층
2. 대화에서 추출된 지식을 담는 위키 계층
3. 필요할 때 더 서술적으로 다시 풀어쓴 블로그/에세이 계층
4. 이 전체를 일관되게 유지하기 위한 운영 규칙과 템플릿

## 추천 저장소 구조

```text
/inbox
  아직 정리되지 않은 대화 원문 또는 메모
/archive
  정리 완료된 원본 대화 기록
/library
  /concepts
  /topics
  /sessions
  /syntheses
  /questions
  index.md
  log.md
/writing
  /posts
/system
  guide.md
  templates/
```

## 계층 설명

### 1. Inbox

- 오늘 나눈 대화 원문
- 아직 정리되지 않은 메모
- 임시로 떨궈두는 수집 공간

### 2. Archive

- 정리 완료 후 보존하는 원본 대화 기록
- 수정하지 않는 원본 계층

### 3. Library

이 저장소의 중심이다. 대화에서 추출된 지식은 여기로 들어간다.

주요 하위 분류:
- `concepts/`: 개념 단위 문서
- `topics/`: 더 넓은 주제 문서
- `sessions/`: 특정 날짜나 대화 세션에서 무엇을 배웠는지 정리
- `syntheses/`: 여러 문서를 엮어 만든 종합 문서
- `questions/`: 아직 답이 열려 있거나 앞으로 탐구할 질문

### 4. Writing

- library의 내용을 바탕으로 만든 블로그형 글
- 복습용 에세이
- 특정 주제를 더 매끄럽게 설명한 글

### 5. System

- 이 도서관을 앞으로 어떤 규칙으로 유지할지 적어 둔 운영 문서
- 템플릿, 명명 규칙, 링크 규칙, 업데이트 규칙

## 핵심 문서 타입

### Session Note

한 번의 대화 세션에서 무엇을 배웠는지 정리한다.

포함 내용:
- 날짜
- 대화 주제
- 핵심 질문
- 핵심 배움
- 새로 생긴 개념/주제 링크
- 이후 파고들 질문

### Concept Note

하나의 개념을 따로 정리한다.

포함 내용:
- 한 줄 정의
- 자세한 설명
- 왜 중요한지
- 관련 개념
- 예시
- 이 개념이 등장한 세션들

### Topic Note

여러 개념을 묶는 넓은 주제를 다룬다.

포함 내용:
- 주제 개요
- 핵심 하위 개념
- 자주 생기는 오해
- 관련 세션 및 종합 문서

### Synthesis Note

여러 세션이나 개념을 엮어서 새 통찰을 만든다.

포함 내용:
- 핵심 논지
- 연결된 개념들
- 충돌하는 관점
- 아직 확실하지 않은 점

### Question Note

당장 답하지 못했지만 가치 있는 질문을 따로 쌓는다.

포함 내용:
- 질문
- 왜 중요한지
- 지금까지 얻은 단서
- 다음에 볼 문서/세션 링크

### Writing Post

도서관의 내용을 더 읽기 좋은 글로 바꾼다.

포함 내용:
- 하나의 명확한 주제
- 배경 설명
- 핵심 주장
- 예시와 비유
- 관련 library 문서 링크

## 주요 운영 흐름

### 흐름 1. 오늘 대화를 도서관에 반영

1. 오늘 대화 내용을 `inbox/` 또는 session raw로 저장한다.
2. 대화에서 핵심 질문, 배움, 개념, 열린 질문을 추출한다.
3. `library/sessions/`에 세션 노트를 만든다.
4. 관련 `concepts/`, `topics/`, `syntheses/`, `questions/` 문서를 생성 또는 업데이트한다.
5. `library/index.md`를 갱신한다.
6. `library/log.md`에 오늘 무엇이 추가/수정되었는지 남긴다.

### 흐름 2. 특정 주제를 다시 복습

1. 사용자가 주제를 말한다.
2. LLM은 library 안의 관련 문서를 읽는다.
3. 필요한 경우 여러 문서를 종합해 새 synthesis를 만든다.
4. 결과가 장기적으로 유용하면 library 또는 writing에 다시 저장한다.

### 흐름 3. 대화 중 나온 좋은 설명을 글로 승격

1. 특정 답변이나 통찰이 재사용 가치가 높다고 판단한다.
2. library 기반으로 더 다듬은 글을 `writing/posts/`에 저장한다.
3. 원래 개념/주제 문서에서 해당 글로 링크한다.

### 흐름 4. 주기적 정리 / 린트

1. 고아 문서를 찾는다.
2. 중복 개념을 찾는다.
3. 링크가 약한 문서를 찾는다.
4. 최근 세션이 반영되지 않은 핵심 주제를 찾는다.
5. 아직 답하지 못한 중요한 질문을 추린다.

## 기능 요구사항

### FR1. 저장소가 곧 산출물이어야 한다

핵심 결과는 실행 중 상태가 아니라 이 폴더 안의 파일이어야 한다.

### FR2. 원본과 지식 계층이 분리되어야 한다

원본 대화 기록과 정리된 지식 문서는 구분되어야 한다.

### FR3. 반복 대화가 누적 업데이트로 이어져야 한다

같은 주제의 새 대화는 가능한 한 기존 문서를 풍성하게 만드는 방향으로 반영되어야 한다.

### FR4. 문서 간 탐색이 쉬워야 한다

개별 문서는 서로 링크되고, `index.md`를 통해 전체 구조를 파악할 수 있어야 한다.

### FR5. 대화 출처를 추적할 수 있어야 한다

각 주요 문서는 어떤 세션/대화에서 유래했는지 확인 가능해야 한다.

### FR6. 질문도 자산으로 남아야 한다

아직 해결되지 않은 질문도 library 안의 중요한 자산으로 저장되어야 한다.

### FR7. 좋은 답변은 글로 재사용 가능해야 한다

질문에 대한 답변 중 장기 가치가 있는 것은 writing 문서로 승격 가능해야 한다.

## 성공 기준

- 사용자가 대화로 공부한 내용을 며칠 뒤에도 이 폴더 안에서 읽고 복습할 수 있다.
- 같은 주제로 여러 번 대화할수록 library가 더 풍성해진다.
- 문서가 단순 대화 로그가 아니라 정리된 지식처럼 읽힌다.
- 중요한 질문, 개념, 주제, 종합 문서가 서로 연결된다.
- 사용자가 "이 폴더가 내 지식 도서관이다"라고 느낄 수 있다.

## 구현 우선순위

### 1단계: 도서관의 뼈대 만들기

- 디렉터리 구조 만들기
- index/log 만들기
- 핵심 템플릿 만들기
- 운영 가이드 만들기

### 2단계: 대화 -> library 반영 흐름 만들기

- 한 세션을 받아 session note 생성
- concept/topic/question 문서 생성 또는 업데이트
- index/log 자동 갱신 규칙 만들기

### 3단계: 복습과 재서술 강화

- synthesis 문서 생성 흐름 만들기
- writing/posts 파생 흐름 만들기

### 4단계: 유지보수 흐름 만들기

- orphan 체크
- 중복 개념 체크
- 미반영 주제 찾기
- 열린 질문 큐 관리

## 실행 가이드

이 계획의 다음 단계 구현은 "앱 개발"이 아니라 "이 저장소를 지식 도서관으로 세팅하는 작업"으로 진행해야 한다. 따라서 다음 구현 단계에서는 UI나 서비스보다 먼저 다음을 만들어야 한다.

- 실제 폴더 구조
- 문서 템플릿
- 운영 가이드
- 첫 세션 ingest 예시

## 후속 결정 포인트

- 기본 문서 언어를 한국어로 둘지
- 세션 하나당 raw 원문을 얼마나 자세히 보관할지
- 어떤 질문을 `questions/`로 승격할지 기준을 둘지
- `writing/`은 수동 생성 중심으로 둘지, 반자동 승격 규칙을 둘지
