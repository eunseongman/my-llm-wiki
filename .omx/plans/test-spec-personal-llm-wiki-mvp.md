# 테스트 스펙: 대화 기반 개인 지식 도서관

## 테스트 목표

이 저장소가 "앱 소스코드"가 아니라 실제 지식 도서관으로 동작하는지 검증한다. 가장 중요한 기준은 대화 내용이 단순 로그가 아니라, 재독 가능한 지식 문서로 축적되는지다.

## 핵심 검증 질문

1. 한 번의 대화가 session note와 관련 지식 문서로 정리되는가?
2. 같은 주제로 다시 대화하면 기존 문서가 보강되는가?
3. 개념, 주제, 종합, 질문 문서가 서로 연결되는가?
4. 나중에 사용자가 이 폴더를 직접 읽으며 복습할 수 있는가?
5. 좋은 설명이 writing 글로 자연스럽게 파생되는가?

## 검증 수준

### 구조 검증

- `inbox/`, `archive/`, `library/`, `writing/`, `system/` 구조가 존재한다
- `library/index.md`, `library/log.md`가 존재한다
- session, concept, topic, synthesis, question, writing용 템플릿이 존재한다

### 문서 생성 검증

- 하나의 대화 입력으로 `library/sessions/` 문서가 생성된다
- 같은 입력에서 하나 이상의 concept 또는 topic 문서가 생성된다
- 질문이 남는 경우 `library/questions/` 문서가 생성되거나 갱신된다
- `library/index.md`에 새 문서가 반영된다
- `library/log.md`에 작업 기록이 남는다

### 누적 업데이트 검증

- 같은 주제의 두 번째 대화를 반영하면 기존 concept/topic 문서가 보강된다
- 이미 있는 개념이 불필요하게 중복 생성되지 않는다
- 새 세션이 기존 synthesis나 question 문서에도 연결된다

### 재독성 검증

- 생성된 문서가 대화 로그 덤프처럼 보이지 않는다
- 각 문서가 "나중에 다시 읽기 위한 문서"처럼 읽힌다
- 중요한 개념과 질문이 명확히 드러난다

### 파생 글 검증

- 가치 있는 설명이나 종합 내용을 `writing/posts/`로 저장할 수 있다
- writing 문서는 관련 library 문서를 다시 링크한다
- writing 문서는 위키 문서를 대체하지 않고 파생물로 유지된다

## 권장 테스트 시나리오

### 시나리오 A: 단일 학습 세션 정리

- 입력: 사용자가 하나의 개념을 배우는 대화
- 기대 결과:
  - session note 1개 생성
  - concept note 1개 이상 생성
  - topic 또는 question 문서 0~1개 이상 생성
  - index/log 갱신

### 시나리오 B: 같은 주제 후속 대화

- 입력: 같은 개념을 더 깊게 파고드는 두 번째 대화
- 기대 결과:
  - 새 session note 생성
  - 기존 concept/topic 문서 업데이트
  - 관련 synthesis 또는 question 문서 연결 강화

### 시나리오 C: 여러 세션 종합

- 입력: 기존 library를 바탕으로 한 복습 질문
- 기대 결과:
  - 관련 문서를 읽고 synthesis note 생성 또는 갱신
  - 이후 복습에 쓸 가치가 있으면 문서로 저장

### 시나리오 D: 글 승격

- 입력: 잘 정리된 concept/topic/synthesis 묶음
- 기대 결과:
  - `writing/posts/`에 글 생성
  - 원본 library 문서와 상호 링크 유지

## 실패 케이스

- 대화에서 뽑아낸 문서가 사실상 대화 복붙에 그치는 경우
- 같은 개념이 비슷한 이름으로 여러 번 생기는 경우
- session note만 쌓이고 concept/topic 문서로 승격이 안 되는 경우
- question 문서가 만들어지지 않아 열린 질문이 사라지는 경우
- writing 문서가 library와 단절되는 경우

## 수동 리뷰 체크리스트

- 이 문서를 몇 주 뒤 다시 읽어도 이해 가능한가?
- "무엇을 배웠는지"가 분명한가?
- "다음에 무엇을 더 보면 좋은지"가 보이는가?
- 문서 간 연결을 따라가며 지식을 넓힐 수 있는가?
- 이 저장소가 실제로 도서관처럼 느껴지는가?

## 구현 완료 기준

- 저장소 구조가 도서관 운영 관점으로 정리되어 있다
- 하나의 대화를 반영해 session/concept/topic/question 중 필요한 문서가 생성된다
- 같은 주제의 후속 대화가 기존 지식을 보강한다
- index/log가 지속적으로 갱신된다
- writing 파생 흐름이 최소 1회 동작한다
